# 嵌入式烧录失败 / 调试器连不上 — 系统排查提示词

将此提示词粘贴给 AI，适用于"代码编译通过，但烧录失败、调试器无法连接、目标芯片无响应"的场景。

---

## 触发条件

出现以下任一现象时触发此规则：
- 烧录中途报错（command mismatch / timeout / SWD fault / no ACK）
- 调试器能枚举但连不上目标芯片
- `Could not find MEM-AP` / `Debug AP not available`
- 芯片看起来像"被锁住了"
- 之前能烧，现在突然不能烧

---

## 核心规则

**烧录失败 ≠ 代码问题。在改动任何代码之前，先按以下顺序逐层排查：**

### 第一层：工具链兼容性

| 检查项 | 操作方法 |
|------|---------|
| 调试器协议兼容 | 换一个烧录工具试试（OpenOCD → pyOCD / JLinkExe / 芯片厂商专用工具），不同工具对同一调试器的兼容性不同 |
| 传输模式兼容 | 如果是 CMSIS-DAP，检查是 HID 还是 Bulk 模式；如果当前模式不稳定，强制切换到另一种 |
| 频率降低 | SWD 频率从 500kHz → 100kHz → 50kHz 逐步下降测试，无线调试器通常上限很低 |
| 烧录后保持暂停 | 加 `halt` 模式烧录，避免烧完立即运行 → 电机/外设启动产生噪声干扰下次连接 |

### 第二层：硬件连接

| 检查项 | 排查方法 |
|------|---------|
| **nRST / NRST 接线** | 先拔掉 nRST 线试试。接错会持续拉低复位 → 芯片永远无法响应 SWD → 看起来像"锁了"。正常不接也可以烧录 |
| SWDIO/SWCLK 是否短路或断路 | 用万用表量通断，两端都要量 |
| GND 共地 | 调试器和目标板必须共地，否则 SWD 信号无参考电平 |
| VTref 检测电压 | 确认调试器能读到目标板电压（通常需 3.3V），电压不对可能是供电问题 |
| 线缆长度 | SWD 线越短越好（<10cm），线长了引入噪声和信号反射 |

### 第三层：供电和噪声

| 检查项 | 排查方法 |
|------|---------|
| 烧录时断开电机/大功率外设 | 电机启动瞬间的电流浪涌会拉低电源、引入地噪声 → 调试器丢包 → 烧录失败。烧录时让电机悬空或断开电机驱动电源 |
| 目标板供电稳定 | 万用表测 MCU 的 VCC 和 GND 间电压是否稳定在额定值 |
| 无线调试器两端蓝灯 | 发射端和接收端都必须常亮，闪烁或不亮 = 链路中断 |

### 第四层：芯片状态

| 检查项 | 判断和应对 |
|------|---------|
| `Could not find MEM-AP` | **不等于芯片被锁**。先检查 nRST 是否被拉低、供电是否正常。可能只是上一次烧录中途失败导致芯片处于异常状态 |
| `DPIDR` 能读到吗？ | 能读到 `DPIDR 0x6ba02477`（MSPM0）/ `0x2BA01477`（STM32）→ SWD 物理链路是好的，问题在芯片状态而非接线 |
| `nRESET = 0` 持续为低？ | 拔掉 nRST 线再试。如果恢复为 1 → 就是接线问题。如果仍为 0 → 检查芯片硬件复位电路 |
| 完全断电重启 | 拔掉所有电源（USB、电池、调试器），等 5 秒，重新上电（先开目标板，再插调试器） |

## 排查决策树

```
编译通过
  │
  └→ 烧录失败
        │
        ├→ 换烧录工具（OpenOCD → pyOCD → 厂商工具）→ 能烧？→ ✅ 工具链问题
        │   └→ 还是不行
        │       │
        ├→ 降 SWD 频率（500k → 100k → 50k）→ 能烧？→ ✅ 频率/链路质量问题
        │   └→ 还是不行
        │       │
        ├→ 拔掉 nRST 线 → 能烧？→ ✅ 复位线接错
        │   └→ 还是不行
        │       │
        ├→ 断开电机等大功率外设供电 → 能烧？→ ✅ 电源噪声干扰
        │   └→ 还是不行
        │       │
        ├→ 目标板完全断电 5 秒重新上电 → 能烧？→ ✅ 芯片异常状态已恢复
        │   └→ 还是不行
        │       │
        └→ 用调试器只读 DPIDR → 能读到？
              │
              ├→ 能 → MEM-AP 不可见 ≠ 锁芯片，可能是安全/电源 AP 隐藏，尝试 Mass Erase 恢复
              └→ 不能 → SWDIO/SWCLK 断线或短路，查接线
```

---

## 交付格式要求

对每个排查步骤，必须输出：

| 步骤 | 操作 | 预期结果 | 实际结果 | 结论 |
|------|------|---------|---------|------|
| 1 | ... | ... | ... | ✅ 通过 / ❌ 不通过 / ⚠️ 不确定 |

一旦某个步骤定位到问题，立即停止后续排查，给出修复方案。不要把所有步骤做完再总结。
