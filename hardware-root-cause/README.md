# 硬件根因分析 — AI 提示词

版本：v1  
来源：I2C 云台 6 小时调试 + PID 循迹 AO1/AO2 接反实战

---

## 场景

软件逻辑被 AI 反复确认无误、参数调到极限、但现象依然不对。强制 AI 穷举硬件故障并做现象对照。

---

## 快速使用

```
我已经反复确认软件逻辑没有问题，参数也调整到极限了，现象依然没变。
现在请按照硬件根因分析规则，穷举所有可能的硬件故障，
并对每个故障模拟其产生的现象，与我的实验观察做对比。

我的实验现象是：[描述你的现象]
我的硬件架构是：[简述 MCU、传感器、电机驱动等]
```
